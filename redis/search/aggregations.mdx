---
title: Aggregations
---

Aggregations let you compute analytics over your indexed data — metrics like averages, sums, and statistics, as well as bucket-based groupings like terms, ranges, and histograms.

Use `index.aggregate()` to run aggregation queries. You can optionally filter documents before aggregating.

<Tabs>

<Tab title="TypeScript">
```ts
const result = await index.aggregate({
  filter: { category: "electronics" },
  aggregations: {
    avg_price: { $avg: { field: "price" } },
  },
});
```
</Tab>

<Tab title="Python">
```python
result = index.aggregate(
    filter={"category": "electronics"},
    aggregations={"avg_price": {"$avg": {"field": "price"}}},
)
```
</Tab>

<Tab title="Redis CLI">
```bash
SEARCH.AGGREGATE products '{"category": "electronics"}' '{"avg_price": {"$avg": {"field": "price"}}}'
```
</Tab>

</Tabs>

Each aggregation is defined with an **alias** (the key you choose for the result) and an **operator** that specifies what to compute.

---

## Metric Aggregations

Metric aggregations compute a single numeric value across matching documents.

All metric aggregations return `{ "value": <number> }`.

The optional `missing` parameter specifies a fallback value for documents where the field is absent. It defaults to `0`.

### $avg

Computes the average value of a numeric field.

<Tabs>

<Tab title="TypeScript">
```ts
const result = await index.aggregate({
  aggregations: {
    avg_price: { $avg: { field: "price", missing: 0 } },
  },
});
// → { avg_price: { value: 40 } }
```
</Tab>

<Tab title="Python">
```python
result = index.aggregate(
    aggregations={"avg_price": {"$avg": {"field": "price", "missing": 0}}}
)
# → {"avg_price": {"value": 40}}
```
</Tab>

<Tab title="Redis CLI">
```bash
SEARCH.AGGREGATE products '{}' '{"avg_price": {"$avg": {"field": "price", "missing": 0}}}'
```
</Tab>

</Tabs>

## Statistical Aggregations

### $stats

Returns count, min, max, sum, and avg for a numeric field in a single aggregation.

<Tabs>

<Tab title="TypeScript">
```ts
const result = await index.aggregate({
  aggregations: {
    price_stats: { $stats: { field: "price", missing: 0 } },
  },
});
// → { price_stats: { count: 9, min: 0, max: 80, sum: 360, avg: 40 } }
```
</Tab>

<Tab title="Python">
```python
result = index.aggregate(
    aggregations={"price_stats": {"$stats": {"field": "price", "missing": 0}}}
)
# → {"price_stats": {"count": 9, "min": 0, "max": 80, "sum": 360, "avg": 40}}
```
</Tab>

<Tab title="Redis CLI">
```bash
SEARCH.AGGREGATE products '{}' '{"price_stats": {"$stats": {"field": "price", "missing": 0}}}'
```
</Tab>

</Tabs>

### $extendedStats

Returns everything `$stats` returns, plus variance, standard deviation, and standard deviation bounds.

The optional `sigma` parameter controls the bounds width (defaults to 2).

<Tabs>

<Tab title="TypeScript">
```ts
const result = await index.aggregate({
  aggregations: {
    price_extended: {
      $extendedStats: { field: "price", sigma: 2, missing: 0 },
    },
  },
});
// → {
//   price_extended: {
//     count: 9, min: 0, max: 80, avg: 40, sum: 360,
//     sumOfSquares: ..., variance: ..., stdDeviation: ...,
//     stdDeviationBounds: { upper: ..., lower: ... }
//   }
// }
```
</Tab>

<Tab title="Python">
```python
result = index.aggregate(
    aggregations={"price_extended": {"$extendedStats": {"field": "price", "sigma": 2, "missing": 0}}}
)
# → {"price_extended": {"count": 9, "min": 0, "max": 80, "avg": 40, "sum": 360,
#     "sumOfSquares": ..., "variance": ..., "stdDeviation": ...,
#     "stdDeviationBounds": {"upper": ..., "lower": ...}}}
```
</Tab>

<Tab title="Redis CLI">
```bash
SEARCH.AGGREGATE products '{}' '{"price_extended": {"$extendedStats": {"field": "price", "sigma": 2, "missing": 0}}}'
```
</Tab>

</Tabs>

### $percentiles

Computes percentile values for a numeric field.

By default, results are **keyed** — returned as a dictionary mapping percentile to value.
Set `keyed: false` to get an array of `{ key, value }` pairs instead.

<Tabs>

<Tab title="TypeScript">
```ts
// Keyed (default)
const result = await index.aggregate({
  aggregations: {
    price_pct: {
      $percentiles: { field: "price", percents: [50, 95, 99], missing: 0 },
    },
  },
});
// → { price_pct: { values: { "50.0": 40, "95.0": 76, "99.0": 79.2 } } }

// Unkeyed
const result2 = await index.aggregate({
  aggregations: {
    price_pct: {
      $percentiles: { field: "price", percents: [50, 95], keyed: false },
    },
  },
});
// → { price_pct: { values: [{ key: 50, value: 40 }, { key: 95, value: 76 }] } }
```
</Tab>

<Tab title="Python">
```python
# Keyed (default)
result = index.aggregate(
    aggregations={"price_pct": {"$percentiles": {"field": "price", "percents": [50, 95, 99], "missing": 0}}}
)
# → {"price_pct": {"values": {"50.0": 40, "95.0": 76, "99.0": 79.2}}}

# Unkeyed
result = index.aggregate(
    aggregations={"price_pct": {"$percentiles": {"field": "price", "percents": [50, 95], "keyed": False}}}
)
# → {"price_pct": {"values": [{"key": 50, "value": 40}, {"key": 95, "value": 76}]}}
```
</Tab>

<Tab title="Redis CLI">
```bash
# Keyed (default)
SEARCH.AGGREGATE products '{}' '{"price_pct": {"$percentiles": {"field": "price", "percents": [50, 95, 99], "missing": 0}}}'

# Unkeyed
SEARCH.AGGREGATE products '{}' '{"price_pct": {"$percentiles": {"field": "price", "percents": [50, 95], "keyed": false}}}'
```
</Tab>

</Tabs>

## Bucket Aggregations

Bucket aggregations group documents into buckets based on field values, ranges, or intervals.

### $terms

Groups documents by field value. Returns buckets with `key` and `docCount`.

Use `size` to limit the number of buckets returned.

<Tabs>

<Tab title="TypeScript">
```ts
const result = await index.aggregate({
  aggregations: {
    by_category: { $terms: { field: "categoryId", size: 10 } },
  },
});
// → {
//   by_category: {
//     buckets: [{ key: 2, docCount: 3 }, { key: 1, docCount: 2 }, ...],
//     sumOtherDocCount: 0,
//     docCountErrorUpperBound: 0
//   }
// }
```
</Tab>

<Tab title="Python">
```python
result = index.aggregate(
    aggregations={"by_category": {"$terms": {"field": "categoryId", "size": 10}}}
)
# → {"by_category": {"buckets": [{"key": 2, "docCount": 3}, ...],
#     "sumOtherDocCount": 0, "docCountErrorUpperBound": 0}}
```
</Tab>

<Tab title="Redis CLI">
```bash
SEARCH.AGGREGATE products '{}' '{"by_category": {"$terms": {"field": "categoryId", "size": 10}}}'
```
</Tab>

</Tabs>

### $range

Groups documents into buckets by numeric ranges. Each range has optional `from` (inclusive) and `to` (exclusive) bounds.

<Tabs>

<Tab title="TypeScript">
```ts
const result = await index.aggregate({
  aggregations: {
    price_tiers: {
      $range: {
        field: "price",
        ranges: [{ to: 30 }, { from: 30, to: 60 }, { from: 60 }],
      },
    },
  },
});
// → {
//   price_tiers: {
//     buckets: [
//       { key: "*-30", docCount: 3, to: 30 },
//       { key: "30-60", docCount: 3, from: 30, to: 60 },
//       { key: "60-*", docCount: 3, from: 60 }
//     ]
//   }
// }
```
</Tab>

<Tab title="Python">
```python
result = index.aggregate(
    aggregations={
        "price_tiers": {
            "$range": {
                "field": "price",
                "ranges": [{"to": 30}, {"from": 30, "to": 60}, {"from": 60}],
            }
        }
    }
)
# → {"price_tiers": {"buckets": [{"key": "*-30", "docCount": 3, "to": 30}, ...]}}
```
</Tab>

<Tab title="Redis CLI">
```bash
SEARCH.AGGREGATE products '{}' '{"price_tiers": {"$range": {"field": "price", "ranges": [{"to": 30}, {"from": 30, "to": 60}, {"from": 60}]}}}'
```
</Tab>

</Tabs>

### $facet

Groups documents by hierarchical `/`-delimited paths. Only works on fields declared as [FACET](/redis/search/schema-definition#facet-fields) in the schema.

| Parameter | Description | Default |
|-----------|-------------|---------|
| `field` | The FACET field to aggregate on | Required |
| `path` | The root path to start from | Required |
| `depth` | How many levels deep to traverse | `1` |
| `size` | Maximum number of children per level | `10` |
| `minDocCount` | Minimum documents for a bucket to appear | `1` |
| `order` | Sort order for buckets | `{ count: "desc" }` |

<Tabs>

<Tab title="TypeScript">
```ts
const result = await index.aggregate({
  aggregations: {
    by_cat: {
      $facet: {
        field: "category",
        path: "/category",
        depth: 1,
        size: 10,
        minDocCount: 1,
        order: { count: "desc" },
      },
    },
  },
});
// → {
//   by_cat: {
//     path: "/category",
//     sumOtherDocCount: 0,
//     children: [
//       { docCount: 2, path: "/category/books", sumOtherDocCount: 0 },
//       { docCount: 2, path: "/category/electronics", sumOtherDocCount: 0 }
//     ]
//   }
// }
```
</Tab>

<Tab title="Python">
```python
result = index.aggregate(
    aggregations={
        "by_cat": {
            "$facet": {
                "field": "category",
                "path": "/category",
                "depth": 1,
                "size": 10,
                "minDocCount": 1,
                "order": {"count": "desc"},
            },
        },
    },
)
# → {"by_cat": {"path": "/category", "sumOtherDocCount": 0,
#     "children": [
#       {"docCount": 2, "path": "/category/books", "sumOtherDocCount": 0},
#       {"docCount": 2, "path": "/category/electronics", "sumOtherDocCount": 0}
#     ]}}
```
</Tab>

<Tab title="Redis CLI">
```bash
SEARCH.AGGREGATE products '{}' '{"by_cat": {"$facet": {"field": "category", "path": "/category", "depth": 1, "size": 10, "minDocCount": 1, "order": {"count": "desc"}}}}'
```
</Tab>

</Tabs>

## Nested Sub-Aggregations

Bucket aggregations (`$terms`, `$range`, `$histogram`) support `$aggs` to compute metrics inside each bucket.

<Tabs>

<Tab title="TypeScript">
```ts
const result = await index.aggregate({
  aggregations: {
    by_category: {
      $terms: { field: "categoryId" },
      $aggs: {
        avg_price: { $avg: { field: "price" } },
        min_price: { $min: { field: "price" } },
      },
    },
  },
});
// → {
//   by_category: {
//     buckets: [
//       {
//         key: 2,
//         docCount: 3,
//         avg_price: { value: 50 },
//         min_price: { value: 20 }
//       },
//       ...
//     ]
//   }
// }
```
</Tab>

<Tab title="Python">
```python
result = index.aggregate(
    aggregations={
        "by_category": {
            "$terms": {"field": "categoryId"},
            "$aggs": {
                "avg_price": {"$avg": {"field": "price"}},
                "min_price": {"$min": {"field": "price"}},
            },
        },
    },
)
# → {"by_category": {"buckets": [
#     {"key": 2, "docCount": 3, "avg_price": {"value": 50}, "min_price": {"value": 20}},
#     ...]}}
```
</Tab>

<Tab title="Redis CLI">
```bash
SEARCH.AGGREGATE products '{}' '{"by_category": {"$terms": {"field": "categoryId"}, "$aggs": {"avg_price": {"$avg": {"field": "price"}}, "min_price": {"$min": {"field": "price"}}}}}'
```
</Tab>

</Tabs>

## Filtering

You can filter documents before aggregating using the same filter syntax as [queries](/redis/search/querying).

<Tabs>

<Tab title="TypeScript">
```ts
// Only aggregate electronics products

  filter: { category: "electronics" },
  aggregations: {
    avg_price: { $avg: { field: "price" } },
    by_brand: {
      $terms: { field: "brand" },
      $aggs: {
        max_price: { $max: { field: "price" } },
      },
    },
  },
});
```
</Tab>

<Tab title="Python">

result = index.aggregate(

</Tab>

## Multiple Aggregations

```ts

    by_category: { $terms: { field: "categoryId", size: 5 } },
    price_ranges: {
      $range: {
        field: "price",
        ranges: [{ to: 50 }, { from: 50 }],
      },
    },
  },
});
```

result = index.aggregate(
